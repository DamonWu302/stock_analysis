# 实时执行计划设计方案

## 1. 目标

将当前已经通过回测收敛出来的“收益优先”参数，落成一套可每日更新的实时执行计划系统，用于输出：

- 今日买入候选
- 今日卖出提示
- 今日减仓提示
- 当前持仓与实时仓位
- 建议总仓位与单票仓位

该系统的定位不是自动下单，而是“策略执行台”与“实时建议层”。

## 2. 当前默认执行模板

实时模板先使用“收益优先”模板作为默认版本，仓位规则定为：

- 最多持有：3 只
- 单票最大仓位：50%
- 总仓位上限：100%
- 新开仓按剩余资金 / 新开仓数量等权分配
- 但单票仍受 50% 上限约束

## 3. 数据来源

实时执行计划建议优先复用现有数据层：

- `daily_score`
  - 提供总分、分项分、信号、摘要
- `daily_factor`
  - 提供 MA、ATR、CMF、MFI、prior_20_high 等
- `market_snapshot`
  - 提供名称、涨跌幅、板块、成交额等横截面信息
- `price_history`
  - 提供开高低收和历史滚动窗口
- `benchmark_history`
  - 提供大盘过滤和大盘共振判断

## 4. 建议新增表

### 4.1 `strategy_plan_run`

用途：
- 记录某个交易日的策略计划批次

建议字段：
- `id`
- `trade_date`
- `template_key`
- `template_name`
- `config_json`
- `score_version`
- `market_status`
- `market_reason`
- `target_exposure`
- `max_positions`
- `max_single_position`
- `created_at`

说明：
- 一天一条主记录
- `target_exposure` 用于表示建议总仓位，例如 `0.0 / 0.5 / 1.0`

### 4.2 `strategy_plan_stock`

用途：
- 记录某日每只股票的建议动作

建议字段：
- `id`
- `run_id`
- `trade_date`
- `symbol`
- `name`
- `action`
- `priority_rank`
- `target_weight`
- `score_total`
- `score_ma_trend`
- `score_volume_pattern`
- `score_capital_sector`
- `score_breakout`
- `score_hold`
- `score_benchmark`
- `buy_reason`
- `sell_reason`
- `signals_json`
- `metrics_json`
- `created_at`

动作建议：
- `buy`
- `hold`
- `trim`
- `sell`
- `avoid`

说明：
- `metrics_json` 用于存当天触发判断时的关键指标快照
- 这样页面和 AI 都不用再临时重算解释文本

### 4.3 `strategy_position`

用途：
- 保存策略当前持仓和每日更新后的状态

建议字段：
- `id`
- `trade_date`
- `symbol`
- `name`
- `status`
- `shares`
- `cost_price`
- `latest_price`
- `market_value`
- `weight`
- `unrealized_pnl`
- `unrealized_return`
- `hold_days`
- `peak_return`
- `last_action`
- `last_action_reason`
- `plan_run_id`
- `updated_at`

状态建议：
- `open`
- `closing`
- `closed`

说明：
- 这张表建议保存“每日快照”，不要只保存最新一条
- 这样后面可以做持仓历史回看

### 4.4 `strategy_trade_signal`

用途：
- 记录买入、卖出、减仓提示，不等于真实成交

建议字段：
- `id`
- `trade_date`
- `symbol`
- `signal_type`
- `signal_price_ref`
- `target_weight`
- `reason`
- `metrics_json`
- `plan_run_id`
- `created_at`

信号类型建议：
- `buy`
- `trim`
- `sell`

### 4.5 `strategy_setting`

用途：
- 保存当前启用的实时模板和执行参数

建议字段：
- `id`
- `setting_key`
- `setting_value`
- `updated_at`

建议保存：
- 当前启用模板
- 当前总仓位策略
- 是否允许人工覆盖

## 5. 每日计划生成流程

### 5.1 日终生成

建议在收盘后执行：

1. 读取最新交易日 `daily_score / daily_factor / market_snapshot`
2. 读取当前实时模板配置
3. 判断市场过滤是否通过
4. 读取当前策略持仓
5. 逐只持仓做卖出/减仓判断
6. 生成买入候选
7. 结合空余仓位给出目标持仓建议
8. 写入 `strategy_plan_run / strategy_plan_stock / strategy_trade_signal / strategy_position`

### 5.2 市场过滤

实时计划使用“收益优先”模板参数：

- `market_score_filter_min_avg`
- `market_score_filter_min_ma5`
- `market_require_benchmark_above_ma20 = true`

输出：
- `market_status = open / cautious / closed`
- `target_exposure`

建议先用简单三档：
- 强：`1.0`
- 中：`0.5`
- 弱：`0.0`

### 5.3 买入候选生成

按当前收益优先模板执行：

- `buy_strict`
- `buy_momentum`
- 核心命中数过滤
- 低位过滤
- 近 3 日放量滞涨过滤
- 风险限制
- 成交额过滤

最终按：
- `score_total`
- `score_breakout`
- `score_capital_sector`
- `priority_rank`

生成买入候选列表。

### 5.4 卖出与减仓判断

对当前持仓逐只判断：

- `sell_drawdown`
- `sell_time_stop`
- `sell_flip_loss`

输出动作：
- `hold`
- `trim`
- `sell`

### 5.5 仓位分配

规则：
- 最多 3 只
- 单票最多 50%
- 新票按剩余现金 / 新开仓数等权
- 如果等权分配超过 50%，则截到 50%

建议：
- 若已持仓 2 只，再开 1 只时，优先保证组合总仓位不超过目标总仓位

## 6. 页面建议

建议新增一个“策略执行台”页面，至少包含：

- 今日市场状态
- 建议总仓位
- 今日买入候选
- 今日减仓/卖出提示
- 当前持仓表
- 每只持仓的关键指标

当前持仓表建议展示：
- 股票
- 成本价
- 最新价
- 仓位
- 浮盈亏
- 持仓天数
- 峰值收益
- 当前建议动作

## 7. AI 集成建议

AI 提示词可以直接读取：

- 当日 `strategy_plan_run`
- 当日 `strategy_plan_stock`
- 当前 `strategy_position`

这样 AI 输出会更稳定，能直接解释：
- 为什么今天建议买
- 为什么今天建议卖
- 为什么当前建议仓位是 50% 或 100%

## 8. 推荐开发顺序

### 第一步
- 建表
- 先把实时模板和计划批次落库

### 第二步
- 写每日计划生成服务
- 先支持买入候选和卖出提示

### 第三步
- 写策略持仓每日更新服务

### 第四步
- 上页面

### 第五步
- 再接 AI 实时建议
