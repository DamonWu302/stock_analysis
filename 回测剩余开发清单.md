# 回测剩余开发清单

更新时间：2026-03-20

## 当前已完成

- 日频回测主链路已完成
  - 使用 `daily_score / daily_factor / price_history / benchmark_history`
  - `T日收盘信号，T+1日开盘成交`
  - 两档买入、四档卖出
  - 最多持有 `4` 只
  - 无加仓
  - 剩余现金按新开仓数量等权分配
  - 手续费、滑点已接入
- 市场环境过滤已完成
  - 使用全市场平均分和 `5` 日均值做前置过滤
- 回测展示已完成
  - 回测列表
  - 回测详情
  - 策略净值 vs 基准 vs 超额曲线
  - 买卖点标记
  - 持仓区间阴影
  - 悬浮提示
- 回测任务化已完成
  - 后台任务创建
  - 任务状态页
  - 自动轮询刷新

## P0

### 1. 回测任务页自动完成跳转

- 状态：未完成
- 目标：
  - 任务完成后自动跳转到对应回测结果页
  - 可增加 `3~5` 秒倒计时提示
- 原因：
  - 当前任务页虽然会自动刷新，但还需要用户手动点击结果链接

### 2. 回测任务失败原因展示优化

- 状态：未完成
- 目标：
  - 失败时将错误来源明确标记
  - 区分数据缺失、配置错误、数据库写入失败、执行异常
- 原因：
  - 现在失败主要显示原始消息，排查效率还不够高

### 3. 回测状态页阶段更细化

- 状态：部分完成
- 已有：
  - `loading / executing / writing / completed / failed`
- 目标：
  - 增加更明确的文案
  - 可细分为“加载数据 / 执行交易 / 汇总统计 / 写入结果”
- 原因：
  - 当前阶段已经够用，但还可以更利于判断卡点

## P1

### 4. 成交受阻/顺延事件可视化

- 状态：未完成
- 目标：
  - 在净值图下方增加事件标记
  - 例如：
    - `U`：涨停买不到
    - `D`：跌停卖不掉
    - `P`：停牌或缺少开盘价
- 原因：
  - 当前详情页只在摘要中显示统计，不够直观

### 5. 交易记录与图表联动

- 状态：未完成
- 目标：
  - 点击净值图某天时高亮当天交易记录
  - 或点击交易记录时定位到净值图对应日期
- 原因：
  - 现在图和表是分离的，复盘效率一般

### 6. 持仓明细增强

- 状态：部分完成
- 已有：
  - 每日持仓已落库
- 目标：
  - 页面展示每日持仓表
  - 展示持仓权重、浮盈亏、持仓天数
- 原因：
  - 目前详情页重点仍在净值和交易，持仓过程还不够完整

### 7. 回测配置模板化

- 状态：未完成
- 目标：
  - 保存常用回测模板
  - 支持一键复用参数组合
- 原因：
  - 现在每次都要重新勾选规则和参数

## P2

### 8. 更真实的交易约束

- 状态：部分完成
- 已有：
  - 无开盘价不成交
  - 近似涨停买不到
  - 近似跌停卖不掉并顺延
- 目标：
  - 更精细处理一字板
  - 更精细处理停牌恢复
  - 若后续数据支持，可加入真实涨跌停价
- 原因：
  - 当前属于日线近似成交模型，不是完全真实撮合

### 9. 卖出规则继续细化

- 状态：部分完成
- 已有：
  - `CMF < 0`
  - 趋势转弱
  - 硬止损
  - 新鲜度失效
- 目标：
  - 将 `MFI` 回落纳入卖出
  - 增加放量滞涨/放量下跌识别
- 原因：
  - 当前卖出规则已能用，但还偏基础

### 10. 回测统计指标增强

- 状态：部分完成
- 已有：
  - 总收益
  - 基准收益
  - 超额收益
  - 最大回撤
  - 胜率
  - 平均持有天数
- 目标：
  - 盈亏比
  - 月度收益
  - 连续盈利/亏损
  - 换手率分析
  - 收益分布
- 原因：
  - 当前适合基础验证，不足以做更深入策略评估

### 11. 批量参数对比

- 状态：未完成
- 目标：
  - 对比不同买卖规则组合
  - 对比不同过滤阈值、持仓数、手续费参数
- 原因：
  - 现在仍是一轮一轮手动跑，不适合策略调参

4. `P1-5` 交易记录与图表联动

## 备注

- 目前回测已经具备“可运行、可观察、可复盘”的最小闭环
- 后续优先做的是“提高复盘效率”和“提高结果解释性”
- 暂时不建议过早做大规模参数搜索，先把单次回测体验打磨顺
